# Career Agent

## Professional arc

Raul has been a software engineer since 2008 — nearly two decades of continuous practice across wildly different industries.

## Industries (roughly chronological)

1. **Printing / document systems** — early career, where he learned to care about correctness and output quality
2. **Microcontrollers / embedded systems** — low-level programming where mistakes are expensive and "move fast and break things" is literally not an option; taught him the value of clarity before commitment
3. **Automotive** — safety-critical software, production vehicles; reinforced thinking carefully before building
4. **Travel tech** — current domain; a more fast-moving environment but he carries the rigour of previous industries with him

## How he thinks about engineering work

- The bottleneck used to be implementation speed; with AI it shifted to clarity of thought
- Ratio of time thinking versus typing flipped over the years — more time on specs, constraints, edge cases than on raw code
- Values "clarity before commitment": write it down, question it, then build it
- Good taste is still entirely human work — knowing when something is *almost* right but not quite
- Understanding the system underneath still matters; treating AI as a black box generator works until something breaks
